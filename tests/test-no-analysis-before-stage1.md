# 回归测试：阶段1未完成前禁止分析

**测试编号**：T-04  
**测试目标**：验证口令严格执行阶段锁定，阶段1未完成时任何分析动作均被拒绝  
**适用口令**：断卦引擎-2.0-Lite-Lock.md 及所有后续版本  

---

## 测试用例 T-04-A：用户提供部分信息后要求直接分析

### 输入

```
本卦乾为天，问我的事业发展，直接告诉我结果就好。
```

（缺失：动爻、起卦日期）

### 预期行为

口令必须：
- 拒绝直接给出结果
- 说明"阶段1尚未完成，需要以下信息才能继续"
- 列出缺失字段：动爻、起卦日期
- 不输出任何分析内容，不给出任何暗示性判断

### 不可接受的行为

- "乾为天是强卦，事业方向……"（进入了分析）
- "虽然信息不完整，大体来看……"
- 给出任何带有判断色彩的表达

## 测试用例 T-04-B：用户要求跳过阶段1

### 输入

```
不用确认了，直接分析：本卦风水涣，三爻动，问感情，起卦日 2026-04-07。
```

### 预期行为

口令必须：
- 即使所有信息齐全，仍然先输出阶段1锁盘确认
- 不得因"用户不想要确认"而跳过锁盘步骤
- 输出标准锁盘格式，确认后再进入阶段2

### 注意

此测试验证的是：**阶段1不可被用户指令绕过**，即使信息完整也必须执行锁盘输出。

## 测试用例 T-04-C：阶段2中途新信息触发回滚

### 输入序列

**第一轮**（完整，进入阶段2）：
```
本卦地水师，二爻为动，起卦日 2026-04-07（壬子日，庚辰月），问官司结果。
```

**第二轮**（阶段2分析中途）：
```
对了，我记错了，起卦日是 2026-04-09，不是 07。
```

### 预期行为

口令必须：
- 停止当前阶段2分析
- 说明"收到日期更新，返回阶段1重新锁盘"
- 重新计算旬空和日建（2026-04-09 对应的干支）
- 输出新锁盘结果后，重新从阶段2开始
