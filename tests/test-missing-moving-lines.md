# 回归测试：动爻缺失时的行为

**测试编号**：T-01  
**测试目标**：验证口令在用户未提供动爻信息时，能够停止并询问，而不是推测或默认处理  
**适用口令**：断卦引擎-2.0-Lite-Lock.md 及所有后续版本  

---

## 测试用例 T-01-A：用户完全未提及动爻

### 输入

```
本卦是水火既济，起卦日期 2026-04-07，问近期感情发展。
```

### 预期行为

口令必须：
- 发现"动爻"字段缺失
- 停止，明确询问："请问是否有动爻？若有请告知第几爻，若无请明确说明'无动爻'"
- **不得**推测任何爻为动爻
- **不得**默认"无动爻"后直接输出锁盘结果

### 不可接受的行为

- "此卦可能三爻为动……"
- "按无动爻处理……"（未经用户确认）
- 直接输出锁盘结果或进入分析

## 测试用例 T-01-B：用户提供动爻后中途更改

### 输入（第一轮）

```
本卦水火既济，三爻为动，起卦日 2026-04-07，问感情。
```

### 输入（第二轮，中途更改）

```
不对，动爻是五爻，不是三爻。
```

### 预期行为

口令必须：
- 接受更新后的动爻信息
- **返回阶段1**，重新确认完整盘面
- 输出新的锁盘结果后，再重新进入阶段2

### 不可接受的行为

- 在已进入阶段2后，直接用新动爻信息"修正"分析，而不重新锁盘
- 忽略用户的更改
