# 六爻预测口令系统

> 一个以提示词工程为核心的六爻预测规则仓库，强调**严格防幻觉、规则可审计、版本可追溯**。

---

## 项目目标

本仓库不是普通的编程项目，而是一套用于管理六爻预测 AI 口令（Prompt）的规则系统。

核心原则：
- **未知即未知**：没有明确信息，绝不推断
- **不造动爻**：动爻必须由用户提供，不得自行判断
- **不造旬空**：旬空依据起卦日干支计算，不得凭空列出
- **不造变卦**：变卦只在有动爻时存在
- **全程可审计**：每一步分析都需说明依据

---

## 仓库结构

```
/
├── README.md              # 本文件，项目总览
├── changelog.md           # 版本更新记录
│
├── prompts/
│   ├── active/            # 当前使用中的口令版本
│   └── archive/           # 已归档的历史版本
│
├── rules/                 # 六爻核心规则文件
├── cases/                 # 真实或示例预测案例
├── tests/                 # 测试用例（用于验证口令行为）
└── templates/             # 新口令、新案例的模板文件
```

---

## 快速开始

1. 阅读 `rules/core-rules.md` 了解本系统的六爻分析规范
2. 查看 `prompts/active/` 中的最新口令版本
3. 用 `templates/` 中的模板创建新案例或新版本口令
4. 每次修改口令后，在 `changelog.md` 中记录变更

---

## 版本管理规范

- 口令文件命名格式：`v{主版本}.{次版本}-{功能描述}.md`
- 示例：`v1.0-liuyao-core.md`、`v1.2-add-shishen-rules.md`
- 修改前先复制当前版本到 `prompts/archive/`，再编辑新版本
- 每次发布新版本必须更新 `changelog.md`

---

## 防幻觉检查清单

在提交任何口令修改前，确认：

- [ ] 口令中是否包含"如果用户未提供动爻，不得推测动爻"的约束？
- [ ] 口令中是否包含"旬空依据日干支计算，不得自行列出"的约束？
- [ ] 口令中是否要求每步分析都注明依据？
- [ ] 口令是否拒绝在信息不完整时给出确定性结论？
- [ ] 口令是否允许回答"此卦信息不足，无法判断"？

---

## 贡献与迭代

每次迭代建议：
1. 在 `tests/` 中写好预期行为的测试用例
2. 修改 `prompts/active/` 中的口令
3. 用测试用例验证新口令行为
4. 记录到 `changelog.md`
