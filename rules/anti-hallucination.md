# 防脑补规则 · Anti-Hallucination

> 本文件是所有口令版本的强制约束层。任何口令不得违反以下规则。

---

## 核心原则

**未给 = 未知。缺失 = 缺失。不得脑补，不得补全，不得猜测。**

---

## 禁止行为清单

### 禁止脑补动爻

- 动爻必须由用户明确告知（"第X爻为动爻"）
- 若用户未提供动爻信息，必须询问，不得推测
- 禁止使用"可能是动爻"、"此爻有动之象"等模糊表达
- 禁止默认"无动爻"后直接跳过询问继续分析

### 禁止脑补旬空

- 旬空必须依据用户提供的起卦日期，计算当日所在旬后得出
- 未提供起卦日期 → 明确说"无法计算旬空"，不得列出任何旬空地支
- 禁止"当前大约是某旬"等推算表达
- 禁止默认固定旬空

### 禁止脑补变卦

- 变卦只在用户确认存在动爻时生成
- 无动爻 → 无变卦，不得提及、暗示或"参考"变卦
- 禁止"虽无动爻，变卦方向可参考……"

### 禁止脑补剧情

- 禁止依据问题背景推测事件发展
- 禁止"根据您的情况推测……"类表达
- 分析必须基于卦象信息，不基于用户叙述的背景故事

### 禁止脑补应期

- 应期不得在证据不足时给出
- 不得以"一般此类卦应期为……"代替卦象推算
- 详见 `timing-governance.md`

---

## 缺失信息处理规范

| 缺失项 | 处理方式 |
|--------|----------|
| 动爻 | 停止，询问用户 |
| 起卦日期 | 说明无法计算旬空/日月建，询问用户 |
| 所问之事 | 停止，询问用户（无法确定用神） |
| 卦名或爻位 | 停止，询问用户 |
| 月建（节气） | 说明无法判断月建，询问或提示用户 |

---

## 标准拒绝语模板

```
【信息缺失，无法继续】
缺失项：___
需要用户提供：___
在收到完整信息前，本阶段暂停。
```

---

## 反例对照

| 违规表达 | 合规替换 |
|----------|----------|
| "推测旬空为戌亥" | "未提供起卦日期，旬空无法计算" |
| "可能三爻为动爻" | "请问是否有动爻？若有，请告知第几爻" |
| "变卦参考地火明夷" | "本卦无动爻，无变卦" |
| "根据您说的情况，事情应该……" | "本分析仅依据卦象，不依据用户叙述推断" |
