# Patch-E：贷款/放款/授信类专项补丁

**补丁编号**：Patch-E
**来源**：反馈案例——问大额贷款能否成，原断"不成"，反馈"寅月已有一半发放"。
**根因**：规则对"月令转换重启、部分成、分批成、缩额成"的约束不够硬，导致把"当前受压"误判为"终局不成"。
**原则**：最小补丁，只改条文，不改口令名，不改执行总序。

---

## 补丁一：§2C 题型主轴简表 新增条目

**落点**：§2C 题型主轴简表，在"竞标/中标"行之后、"寻物"行之前插入。

**新增条文（可直接粘贴）：**

```
贷款/放款/授信：官＝审批结果/额度是否批｜父＝材料/合同/征信/文书｜财＝资金落袋/实际到账｜兄＝分流/扣减/拆额/竞争占额
```

**防什么误判**：防止贷款类题只套一般审核模板、漏掉"财＝到账"与"兄＝拆额"两条关键取象线，导致只看官不看钱、只看批不看到账。

---

## 补丁二：§15 审核类模板 新增 §15B

**落点**：§15 审核类模板末尾，§15A 之后，新增 §15B。

**新增条文（可直接粘贴）：**

```
§15B 贷款/放款/授信专项
贷款类题在审核模板基础上，须额外遵守以下三条：

一、官爻当前受压≠终局不成
官爻受月克或日克但未绝未墓未破者，只入过程阻滞层，不得直接升级终局利空。
须同时核查：
① 下一月令转换后，官是否得生扶或解克（月令重启窗口）；
② 是否存在动爻生官、合官、回头生官的恢复路径；
③ 父母爻（合同/文书）是否在场且未断。
以上三项命中任一，官的受压只归过程，终局须保留"成但卡/补后成/迟成"路径参与比较。
口诀：当前月克不锁死，先看下月能不能接。

二、月令转换重启必须纳入终局比较
若占时月令克官或克财，但下一月令生扶官或财，且距月令转换≤30日，则：
月令重启路径必须作为终局利成候选纳入比较，不得只按占时月令锁死终局。
若月令重启路径为唯一利成候选，概率上限中高，不得拉满。

三、贷款类终态颗粒度
贷款类铁口过程语不得只切"成/不成"粗颗粒，必须从以下五档中选取：
全额成｜部分成｜分批成｜缩额成｜不成
判定依据：
全额成＝官得力＋财到位＋兄不夺
部分成＝官得力＋兄动分流或财受克减量
分批成＝官得力＋财受阻但有后续解除窗口（月令转换/冲实/合处逢冲）
缩额成＝官得力＋兄旺克财或财弱
不成＝官终局不立（绝/墓/破且无恢复路径）或终止链成立
口诀：批了不等于全到，拆了不等于不批。
```

**防什么误判**：防三类错误——①把官的当月受克直接判终局不成、忽略月令转换重启；②把部分放款误判为完全不成；③终态颗粒度太粗，无法识别分批/缩额等现实常见落地形态。

---

## 补丁三：§2F 四问裁决 补充硬约束

**落点**：§2F 四问裁决，在现有"规则"段落末尾、"四问只拍板"之前插入。

**新增条文（可直接粘贴）：**

```
贷款/金额型审批加裁：
若③"终止机制"答不硬（无不可逆终止动作），但④"最合理成法"可明确答出为补件、降额、分批、延后发放之一，
则不得轻易站死终局不成；
优先回到"成但卡 / 补后成 / 缩额成 / 分批成"路径比较，概率按证据强度定档。
口诀：终止答不硬＋成法答得出＝不许判死。
```

**防什么误判**：防止四问裁决时，仅因"当前有压力"就站死不成，而忽略了现实中贷款审批常见的"补件后批、降额批、分批放"等非终止性结局。

---

## 补丁四：§2I 白话收口 新增语义锁 + §5 允许表达

**落点一**：§2I 白话收口 语义锁表，在"审核"行之后插入。

**新增条文（可直接粘贴）：**

```
贷款/放款:成＝能批下来（含全额/部分/分批/缩额）｜不成＝批不了
```

**落点二**：§2I 白话收口纪律段末尾，补入：

**新增条文（可直接粘贴）：**

```
贷款类白话须与§15B终态颗粒度一致，不得只说"能批/批不了"两档。
允许表达：
能批但额度会打折；
能批但不是一次到位，会分批放；
能批但要先补材料/等下个月；
当前卡着批不顺，但后段有重启窗口，不是死局。
```

**落点三**：【5｜黑名单与允许表达】允许表达段末尾，补入：

**新增条文（可直接粘贴）：**

```
全额成 / 部分成 / 分批成 / 缩额成
能批但打折｜能批但分批｜当前卡但有重启窗口
```

**防什么误判**：防止白话收口只剩"能批/批不了"两档，丢失"部分成/分批成/缩额成/有重启窗口"等现实中最常见的贷款落地形态，使用户误以为"不是全批就是全拒"。

---

## 补丁五：§2G 铁口三行 过程语扩展

**落点**：§2G 铁口三行，过程语允许值行。

**修改前**：

```
过程语：顺成 / 卡成 / 补后成 / 迟成 / 卡败 / 拖败｜原因≤25字｜翻盘≤3
```

**修改后（可直接粘贴）：**

```
过程语：顺成 / 卡成 / 补后成 / 迟成 / 缩额成 / 分批成 / 卡败 / 拖败｜原因≤25字｜翻盘≤3
```

**防什么误判**：防止过程语词表里没有"缩额成/分批成"选项，导致执行时只能选"成"或"不成"，无法表达贷款类最常见的中间态结局。

---

## 配套：§2A 验收清单触发项扩展

**落点**：§2A 验收清单，"命中才展开"列表中"审核/竞标"行。

**修改前**：

```
审核/竞标→终止链检查（竞标/中标并调用§15A）
```

**修改后（可直接粘贴）：**

```
审核/竞标/贷款→终止链检查（竞标/中标并调用§15A；贷款/放款/授信调用§15B）
```

**防什么误判**：防止贷款类题不触发§15B专项路由，仍走一般审核模板而漏检月令重启和终态颗粒度。

---

## 配套：§2C 题型映射扩展

**落点**：§2C 两现时规则之前，题型主轴简表最后一行之后。

**新增条文（可直接粘贴）：**

```
注：贷款/放款/授信类两现财时，按"在场财＝已批/已到账部分"vs"伏财/弱财＝未到/待放部分"分判，不得只取一面。
```

**防什么误判**：防止财爻两现时只取一面，忽略"已到账部分"与"未放部分"的分判需求。

---

## Patch-E 变更摘要

| 序号 | 落点 | 动作 | 防的误判 |
|------|------|------|----------|
| 1 | §2C 题型主轴简表 | 新增贷款行 | 漏取财/兄取象线 |
| 2 | §15B（新增） | 贷款专项三条 | 当前受压≠终局不成；月令重启漏检；终态粗颗粒 |
| 3 | §2F 四问裁决 | 补贷款加裁 | 终止答不硬时误站死不成 |

| 5 | §2G 铁口三行 | 过程语扩词 | 缩额成/分批成无法表达 |

---

**总结：这次改的是规则条文，不是口令名；它会影响推理约束，因此会影响准确率。**

案例溯源：本补丁源自"大额贷款能否成案"反馈复盘，原案核心教训为"官受月克误判终局不成，忽略月令转换重启窗口；终态粗颗粒无法表达部分成/分批成"，特此记录以备后续执行者理解规则语境。
