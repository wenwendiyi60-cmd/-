# 审计清单 · Audit Checklist

> 每次发布新口令版本前，必须逐项检查。全部通过后方可归入 active/。

---

## 一、防脑补检查

- [ ] 口令是否明确禁止在未获用户动爻信息时生成任何动爻内容？
- [ ] 口令是否在用户未提供日期时，禁止列出旬空？
- [ ] 口令是否在无动爻时，完全禁止提及变卦？
- [ ] 口令是否禁止基于用户叙述的背景故事推断结论？
- [ ] 口令是否允许输出"信息不足，无法断定"？

---

## 二、阶段协议检查

- [ ] 阶段1锁盘是否要求收集全部必要字段后才输出锁盘结果？
- [ ] 阶段1输出中是否不含任何分析内容？
- [ ] 阶段2是否按固定顺序执行（用神→旺衰→双面取象→证据分层→四问裁决）？
- [ ] 阶段2是否在阶段1完成前被锁定？
- [ ] 中途补充信息后，口令是否要求返回阶段1重新锁盘？

---

## 三、证据与断语检查

- [ ] 断语是否基于阶段2的四问裁决结果，而非中间推论？
- [ ] 断语是否明确给出结论（事成/难成/尚可/无法断定之一）？
- [ ] 断语是否列出强证据与中证据，并区分层级？
- [ ] 断语是否注明所有不确定项？
- [ ] 口令是否禁止在仅有弱证据时给出确定性结论？

---

## 四、应期检查

- [ ] 口令是否禁止在卦象证据不足时给出应期？
- [ ] 应期是否只来自卦象推算，而非背景故事判断？
- [ ] 详见 `timing-governance.md`

---

## 五、回归测试检查

- [ ] `test-missing-moving-lines.md` 通过？
- [ ] `test-missing-xunkong.md` 通过？
- [ ] `test-static-no-biangua.md` 通过？
- [ ] `test-no-analysis-before-stage1.md` 通过？
- [ ] `test-process-vs-outcome-evidence.md` 通过？

---

## 六、版本管理检查

- [ ] 旧版本是否已复制至 `prompts/archive/`？
- [ ] `changelog.md` 是否已更新？
- [ ] 新文件命名是否符合规范（`v{主}.{次}-{描述}.md`）？
